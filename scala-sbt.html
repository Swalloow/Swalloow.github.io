Scala의 빌드 도구 SBT

08 Jul 2017 on Develop

​

Scala에는 SBT라는 빌드 도구가 있습니다. SBT는 의존성 관리에 Apache ivy를 사용합니다. 심플함을 주장하지만 별로 와닿지는 않습니다. 앞으로 계속 내용을 추가할 예정입니다.

​

SBT로 프로젝트를 생성해보자

SBT를 사용하기 위해서는 당연히 SBT가 설치되어 있어야 합니다. OS X인 경우 HomeBrew로 쉽게 설치할 수 있습니다. brew install sbt

프로젝트 폴더를 생성하고 이동한 다음, sbt 명령어를 통해 sbt-shell로 이동할 수 있습니다. 아래의 명령어를 통해 기본 구조를 생성할 수 있습니다.

그런데 사실 저렇게하면 src 폴더는 제외됩니다. 그래서 아래의 스크립트로 기본 폴더 구조를 생성했습니다.

전체적인 구조는 Maven과 유사합니다.

SBT로 생성한 프로젝트의 루트 경로에 build.sbt가 생성된 것을 확인할 수 있습니다.

​

자주 사용하는 SBT 명령어

  • actions : 사용 가능한 명령 확인
  • clean : target 디렉토리의 생성된 모든 파일을 삭제
  • update : 프로젝트가 사용하는 라이브러리 다운로드
  • compile : 소스코드 컴파일
  • test : 테스트 실행
  • run : 메인 함수를 통해 코드를 실행
  • reload : 빌드 정의 변경 후 재실행
  • console : 스칼라 인터프리터를 실행
  • package : 배포 가능한 jar파일 생성
  • publish-local : 만들어진 jar를 로컬 ivy 캐시에 설치

  • publish : jar를 원격 저장소에 배포 (원격 저장소 설정 필요)

    ​
